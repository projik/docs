dAvePi is a schema-driven backend framework that turns a single configuration file into a fully working production backend. Write one schema, and dAvePi automatically generates REST and GraphQL APIs, an MCP server for AI agents, live Swagger documentation, an admin dashboard, and a typed TypeScript client — all kept in lockstep as your data model evolves.

## What you get from one schema file

Drop a file like this into `schema/versions/v1/`:

```js schema/versions/v1/project.js
module.exports = {
  path: 'project',
  collection: 'project',
  fields: [
    { name: 'userId',  type: String, required: true },
    { name: 'name',    type: String, required: true, searchable: true },
    { name: 'status',  type: String, enum: ['active', 'archived'], default: 'active' },
    { name: 'dueDate', type: Date },
  ],
};
```

And dAvePi instantly provides:

## Key capabilities

- **Multi-tenant by default** — every record is automatically scoped to its owner. One user can never read another's data through any surface.
- **Role-based access control** — document-level and field-level ACL gates applied symmetrically across REST, GraphQL, and MCP.
- **Plugin ecosystem** — switch on auth (magic-link, OAuth, SMS), Stripe billing, Slack/email/SMS notifications, full-text search, file storage, job queues, and Sentry with a single config line.
- **AI agent layer** — run chat agents with personas, durable memory, governed skill runbooks, and per-user profiles on top of your data.
- **Lifecycle hooks & webhooks** — react to create/update/delete events both inside your app and in external systems.
- **State machines** — model business workflows (e.g. `draft → review → published`) and enforce valid transitions automatically.
