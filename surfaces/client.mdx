dAvePi ships a code generator that reads your live schema registry and emits a single, fully-typed TypeScript client file. The same source of truth that drives your server's REST and GraphQL surface also drives your frontend's compile-time types Б─■ a typo on a field name fails `tsc`, not at 4 AM in production.

## Generate the client

Run the generator from your project root. The CLI boots the schema loader exactly as the HTTP server does (without calling `app.listen()`), walks the registry, writes one `.ts` file, and exits.

```bash
npx davepi gen-client --out ./client/davepi.ts
```

Specify a base URL to bake it into the generated factory:

```bash
npx davepi gen-client --out ./client/davepi.ts --base-url https://api.example.com
```

<Note>
Output is deterministic Б─■ schemas are sorted alphabetically, no timestamps are emitted. Running the generator twice against the same registry produces a byte-identical file, so CI diffs stay clean.
</Note>

---

## Set up the generated files

Drop the generated file into your project alongside the hand-written runtime companion, which ships with dAvePi and never needs to be regenerated:

```
src/
  api/
    davepi.ts            # generated Б─■ do not edit by hand
    davepi-runtime.ts    # copy once from dAvePi's client/davepi-runtime.ts
```

## Use it in a frontend app

```ts
import { createDavepiClient } from './api/davepi';

const api = createDavepiClient({
  baseUrl: 'https://api.example.com',
  getToken: () => localStorage.getItem('davepi-token') || '',
});

// List with filtering, pagination, and relation loading
const accounts = await api.account.list({
  filter: { accountName: { $regex: '^Acme' } },
  page: 1,
  perPage: 20,
  include: ['contacts'],          // typed as a literal union Б─■ typos caught at compile time
});

// Create with idempotency
await api.account.create(
  { accountName: 'Acme Inc' },
  { idempotencyKey: crypto.randomUUID() }
);

// Trigger a state-machine transition Б─■ `to` is typed as the declared state literal
await api.quote.transitionStatus(quoteId, 'approved');
```

## What gets generated per schema

For each resource, the generator emits four symbols:

| Symbol | Shape |
|--------|-------|
| `interface <Resource>` | Read shape Б─■ the server response. Includes computed fields (marked `readonly`), file fields typed as `FileMeta`, state-machine fields typed as literal unions, `availableTransitions`, and framework metadata (`createdAt`, `updatedAt`, `deletedAt`). |
| `interface <Resource>Writeable` | Request body shape for POST and PUT. Excludes server-stamped fields (`userId`, `accountId`), computed fields, and file fields. |
| `type <Resource>Include` | Literal union of relation names Б─■ e.g. `'contacts' \| 'primaryContact'` Б─■ or `never` when no relations are declared. |
| `interface <Resource>Client` | Method signatures: `list`, `get`, `create`, `update`, `delete`, plus optional methods depending on schema flags. |

### Optional methods per schema

| Method | Generated when |
|--------|---------------|
| `restore(id)` | `softDelete` enabled (default) |
| `history(id, params?)` | `audit` enabled (default) |
| `search(q, params?)` | Any field has `searchable: true` |
| `<aggregationName>(args)` | Per declared aggregation |
| `<relationName>(id)` | Per relation Б─■ returns array for `hasMany`, single record for `hasOne`/`belongsTo` |
| `transition<Field>(id, to)` | Per state-machine field Б─■ `to` is the literal union of declared states |
| `upload<Field>(id, file)` | Per `type: 'File'` field |
| `fetch<Field>Url(id)` | Per `type: 'File'` field |
| `delete<Field>(id)` | Per `type: 'File'` field |

### Global client type

The generator also emits a top-level `DavepiClient` interface and the `createDavepiClient` factory:

```ts
export interface DavepiClient {
  account: AccountClient;
  contact: ContactClient;
  deal: DealClient;
  // one entry per loaded schema
}

export function createDavepiClient(opts: ApiOptions): DavepiClient;
```

## Example generated snippet

Here is a fragment of what the generator emits for an `account` schema with a `contacts` relation and a `status` state machine:

```ts
// Generated by `davepi gen-client`. Do not edit by hand Б─■
// re-run after a schema change. Pairs with ./davepi-runtime.ts.

export interface Account {
  _id: string;
  userId: string;
  accountName: string;
  industry?: string;
  readonly isActive: boolean;         // computed field
  status: 'active' | 'suspended' | 'closed';
  createdAt: string;
  updatedAt: string;
  deletedAt: string | null;
  availableTransitions?: AvailableTransitions;
  contacts?: Contact[];               // populated via include
}

export interface AccountWriteable {
  accountName: string;
  industry?: string;
  status?: 'active' | 'suspended' | 'closed';
}

export type AccountInclude = 'contacts' | 'primaryContact';

export interface AccountClient {
  list(params?: ListParams<AccountInclude>): Promise<ListResponse<Account>>;
  get(id: string, opts?: { include?: AccountInclude[] }): Promise<Account>;
  create(record: AccountWriteable, opts?: IdempotentCreateOpts): Promise<Account>;
  update(id: string, record: Partial<AccountWriteable>): Promise<Account>;
  delete(id: string): Promise<DeleteResult>;
  restore(id: string): Promise<void>;
  history(id: string, params?: { page?: number }): Promise<ListResponse<AuditEntry>>;
  contacts(id: string): Promise<Contact[]>;
  transitionStatus(id: string, to: 'active' | 'suspended' | 'closed'): Promise<Account>;
}
```

## Filtering on the wire

Filters are passed as a `Record<string, unknown>` and serialised onto the URL using mongo-querystring conventions. The runtime handles the encoding Б─■ you write plain objects:

```ts
api.account.list({
  filter: {
    accountName: { $regex: '^Acme' },
    createdAt:   { $gte: '2025-01-01' },
  },
});
// Б├▓ GET /api/v1/account?accountName={"$regex":"^Acme"}&createdAt={"$gte":"2025-01-01"}
```

## Error handling

The runtime throws `DavepiError` for any non-2xx response. Catch it to branch on typed error codes:

```ts
import { DavepiError } from './api/davepi-runtime';

try {
  await api.quote.transitionStatus(id, 'approved');
} catch (err) {
  if (err instanceof DavepiError && err.code === 'INVALID_TRANSITION') {
    console.log('Allowed transitions:', err.details.allowed);
  }
}
```

`DavepiError` carries:
- `status` Б─■ HTTP status code.
- `code` Б─■ typed string (`VALIDATION`, `NOT_FOUND`, `CONFLICT`, `INVALID_TRANSITION`, `IDEMPOTENCY_CONFLICT`, and others).
- `message` Б─■ human-readable description.
- `details` Б─■ structured payload for typed errors like `INVALID_TRANSITION`.

## The runtime companion

`davepi-runtime.ts` is hand-written and ships once. It provides:

- `ApiOptions` Б─■ `{ baseUrl, getToken, fetch?, headers? }`
- `ListParams<TInclude>`, `ListResponse<T>`, `IdempotentCreateOpts`, `DeleteResult`, `AuditEntry`, `FileMeta`, `AvailableTransitions`
- `DavepiError` Б─■ thrown for non-2xx responses
- `buildHttpClient(opts)` Б─■ the underlying fetch wrapper
- `makeResourceClient(client, config)` Б─■ the factory the generator calls per schema

The runtime has zero runtime imports beyond the global `fetch`. It works in Node Б┴╔ 18, browsers, Cloudflare Workers, Deno, and Bun.

## Re-running after schema changes

Run the generator after every schema change, then let `tsc` surface broken call sites:

```bash
git commit -m "Add Account.region field"
npx davepi gen-client --out ./client/davepi.ts
npm run typecheck     # any code that relied on the old shape lights up red
```

Lock the generated file to your schema map with a CI step:

```yaml
- run: npx davepi gen-client --out client/davepi.ts
- run: git diff --exit-code client/davepi.ts
```

This ensures the committed client is always in lockstep with the schema map Б─■ if someone changes the schema without regenerating the client, the CI step fails.
